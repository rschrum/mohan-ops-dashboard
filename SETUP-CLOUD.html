☁️ Cloud Deployment Guide One-Time Setup

Move the Mohan Ops dashboard to the cloud so it auto-refreshes 24/7 — no computer required.

⚡ How This Works (30-second overview)

🐙
GitHub Repo
Stores the dashboard files. GitHub Actions runs a script every 30 min for free.
⚙️
GitHub Actions
Reads Slack, updates tasks.json, regenerates index.html, commits — all automated.
🌐
Netlify (auto-deploy)
Connected to GitHub — detects every new commit and instantly publishes the updated dashboard.
💰
Total cost: $0. GitHub Actions free tier (public repo = unlimited minutes), GitHub free account, Netlify free tier. Everything here is completely free.

🛠 Setup Steps

1
Create a Slack Bot
~5 minutes · Only needs to be done once
1
Go to api.slack.com/apps
Visit api.slack.com/apps and click Create New App → From Scratch.
2
Name the app and pick your workspace
Name: Mohan Ops Dashboard · Workspace: mohanops · Click Create App.
3
Add OAuth Scopes
In the left sidebar: OAuth & Permissions → Scopes → Bot Token Scopes. Add all 5 of these:
channels:history groups:history im:history channels:read users:read
4
Install the app to your workspace
Scroll up on the OAuth page and click Install to Workspace → Allow. You'll be redirected back to the OAuth page.
5
Copy the Bot Token
On the OAuth page, copy the Bot User OAuth Token. It starts with xoxb-. Save this — you'll need it in Step 3.

⚠️
Keep this token private. Never paste it in Slack, email, or a public file. It goes only in GitHub Secrets (next steps).
2
Create a GitHub Repository
~3 minutes · One-time
1
Create a new repo at github.com
Go to github.com/new. Settings:
• Name: mohan-ops-dashboard
• Visibility: Public (recommended — unlimited free Actions minutes)
• Leave all checkboxes unchecked

Click Create repository.
2
Upload all files from the cloud-deploy folder
On the new (empty) repo page, click uploading an existing file. Drag the entire contents of the cloud-deploy folder into the upload area. Make sure .github/workflows/refresh.yml is included.
💡
Important: The .github folder starts with a dot and may be hidden on Mac. To show hidden files in Finder, press Cmd + Shift + . (period). Make sure to include that folder — it's what tells GitHub to run the script automatically.
Click Commit changes to upload.
3
Add the Slack Token as a Secret
~2 minutes · Keeps the token secure — never in plain text
1
Open repo Settings → Secrets
In your GitHub repo, click Settings (top tab) → Secrets and variables → Actions → New repository secret.
2
Add the secret
• Name: SLACK_BOT_TOKEN (must be exact, all caps)
• Secret: paste your xoxb-... token here

Click Add secret.
4
Connect Netlify to GitHub (auto-deploy)
~3 minutes · This replaces the manual drag-and-drop — Netlify auto-updates every time GitHub gets a new commit
1
Log in to Netlify and create a site from GitHub
Go to app.netlify.com → Add new site → Import an existing project → GitHub. Authorize Netlify to access your repos if prompted.
2
Select your repository
Choose mohan-ops-dashboard from the list.
3
Configure deploy settings
• Branch to deploy: main
• Publish directory: leave blank (or type /)
• Build command: leave blank

Click Deploy site. Netlify will build and give you a URL like random-name.netlify.app.
4
(Optional) Rename the URL
Go to Site settings → General → Site details → Change site name. Set something like mohan-ops-dashboard to get mohan-ops-dashboard.netlify.app.
5
Test It
~2 minutes · Confirm everything is wired up correctly
1
Trigger a manual run in GitHub Actions
In your repo, click the Actions tab → Refresh Dashboard → Run workflow → Run workflow. This fires the script immediately instead of waiting 30 minutes.
2
Watch the run complete
The run takes about 30–60 seconds. Click on it to see the live log. You should see messages like 📡 #all-mohan-ops, 💾 Saved tasks.json, ✅ Refresh complete. If there's a red ❌, click to read the error — most likely the token scope is wrong or the secret name doesn't match exactly.
3
Check Netlify auto-deployed
Go back to app.netlify.com — you should see a new deployment triggered automatically. Visit your Netlify URL and verify the dashboard shows your tasks. Done! 🎉

✅ What you have after setup

  • GitHub Actions runs refresh.py every 30 minutes, 24/7 — no computer needed
  • Netlify auto-deploys within seconds of every GitHub commit — your team always sees the latest
  • Slack commands still work exactly the same — post in #all-mohan-ops and the board updates on the next cycle
  • Local Claude scheduled task can be kept as a backup or disabled — your choice
  • Total monthly cost: $0

🔧 Troubleshooting

ProblemFix
GitHub Actions run fails with missing_scopeGo back to api.slack.com/apps → OAuth & Permissions → add the missing scope → reinstall the app
Run fails with SLACK_BOT_TOKEN not setCheck Settings → Secrets — the name must be exactly SLACK_BOT_TOKEN (all caps, no spaces)
Netlify doesn't auto-deployCheck that Netlify is connected to the correct GitHub repo. Go to Netlify → Site settings → Build & deploy → verify the repo link.
Dashboard shows no tasksThe bot needs to be installed to the workspace and the channels need to exist. Check the Actions run log for error details.
Cron doesn't fire on timeGitHub Actions cron can be delayed by a few minutes under heavy load. This is normal — the schedule is approximate, not exact.
Hidden .github folder not visible on MacIn Finder, press Cmd + Shift + . to show hidden files, then drag the .github folder into the GitHub upload.
